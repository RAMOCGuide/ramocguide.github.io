Ai Tweaker› DRAM Timing Control› Primary Timings / Secondary Timings
Стратегия: Запази Primary Timings на XMP стойностите (30-36-36-66). Вторичните timings имат по-малко влияние върху стабилността и могат да се стягат. Тествай след всяка промяна с 30 мин OCCT Memory.
Навигация: Ai Tweaker → (скролни надолу) → DRAM Timing Control → (Enter) → DIMM Fit → Primary Timings / Secondary Timings. Използвай + и - клавишите за промяна на стойностите. Напиши "Auto" + Enter за връщане към Auto.

При нестабилност
⛔ Задължително: DDR5 грешки могат да корумпират данни без crash. Не ползвай системата за работа без потвърдена стабилност с поне MemTest86 + OCCT Memory.
Препоръчан ред на тестване
1
POST тест — система стартира нормално
Ако не стартира 3 пъти последователно → BIOS auto-reset към defaults. Q-Code C1/C3 = memory training fail.
2
Windows Event Viewer — System log
Провери за Memory errors (Source: MemoryDiagnostics-Results) след първото стартиране.
3
MemTest86 — минимум 2 прохода
Безплатен, bootable USB. При 64GB 1 проход ≈ 1 час. 0 грешки = задължително. Свали от memtest86.com
4
OCCT Memory Linpack — 1 час
Тества под реален OS товар. Следи RAM температура в HWiNFO64 → не трябва да надвишава 50°C. Свали от ocbase.com
5
HWiNFO64 → Memory Errors Counter
Остави да работи 30+ мин под натоварване. DDR5 On-Die ECC коригира единични бит грешки — провери correctable errors counter.
