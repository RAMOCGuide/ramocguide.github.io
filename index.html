Voltage домейни — пълна карта

Memory Controller Voltage (VDD2) BIOS: Memory Controller Voltage

Статичен 1.1V rail. Захранва цифровата DDR PHY физическа подсистема (DDRIO). Това НЕ е System Agent — напълно различен rail. В ASUS BIOS се казва "Memory Controller Voltage". Диапазон: 0.900–1.800V.

Източник: SkatterBencher Arrow Lake MemSS; Shamino "Storming the Gates" ROG Forum

CPU System Agent Voltage (VCCSA) BIOS: CPU System Agent Voltage

Динамичен SVID rail. Захранва аналоговата DDRIO PHY, NGU (NoC) и SoC IP блокове. Споделен с NGU — по-високото от двете стойности печели. За NGU/D2D OC нужен 1.15–1.25V daily. 200S Boost автоматично вдига до 1.2V.

Източник: Shamino "Storming the Gates", OCN page 691, SkatterBencher #87

VDDQTX (вътрешен, bypass mode) Не пипайте при daily OC

Аналогов LVR rail за TX сигнализиране от MC към DIMM слотовете. По default в bypass mode, синхронизиран с VDD2. Не го пипайте освен при екстремен OC.

DRAM VDD / VDDQ BIOS: DRAM VDD Voltage / DRAM VDDQ Voltage

Захранват DRAM чиповете директно. Управляват се от PMIC регулатора на платката. При 1.4V+ е препоръчително да активирате "High DRAM Voltage Mode" — без него горната граница е само 1.435V.

Източник: OCN Arrow Lake thread page 691, community consensus за Z890

Критична разлика от Raptor Lake

Традиционният "CPU SA voltage" при Raptor Lake отговаряше на IMC. При Arrow Lake, Memory Controller Voltage (VDD2) и CPU System Agent (VCCSA) са два напълно различни rails. Объркването им е честа причина за нестабилност при Z890.

Tile архитектура и латентност

Защо Arrow Lake има по-висока памет латентност от Raptor Lake

IMC живее на SoC тайла (TSMC N6), докато CPU ядрата са на отделен Compute тайл (Intel 20A). Комуникацията минава през D2D (Die-to-Die) шина, която добавя ~10–15 ns структурна латентност. Тя е неизбежна и не е грешка в настройките.

Очаквайте AIDA64 ~68–88 ns при Gear 2 + DDR5-6000 — нормално за платформата.

Die идентификация: A-die vs M-die

Практически тест чрез tRFC поведение

Намалявайте tRFC при DDR5-6000 постепенно. Hynix A-die няма да boot или ще crash при тRFC под ~420–450ns. Hynix M-die понася значително по-ниски стойности (под 350ns). Твоите "BB" модули с BB суфикс = A-die.

Специфика при 2×32GB: реален потребителски доклад

Критична находка — OCN потребител с точно 2×32Gb A-die на Z890

Докладва: VDD >1.40V → системата не boot-ва; VDDQ >1.35V → нестабилност при 6800+. Оптималното при push: VDD=1.40V, VDDQ=1.35V.

При DDR5-6000 CL30 (твоят XMP профил): VDD и VDDQ 1.40V са перфектни и напълно безопасни.

High DRAM Voltage Mode — важна корекция

Трябва ли да го активирате?

Да, при ≥1.40V е препоръчително. Без него горната PMIC граница е 1.435V — само 35mV буфер при 1.40V XMP. С активирано High DRAM Voltage Mode (разширява до 2.07V), PMIC регулаторът работи по-комфортно дори при 1.40V. Не го активирайте само ако планирате ≤1.35V.

Източник: OCN Arrow Lake thread page 691, community consensus за Z890

VDDQ специфика за 2×32 A-die: Community докладва VDDQ >1.35V причинява нестабилност при 6800+ MT/s. При DDR5-6000 CL30 — 1.40V е перфектно. При push към 6800 → ако имате issues, намалете VDDQ до 1.35V докато VDD остава 1.40V.

Ограничения — критично важно

200S Boost заключва OC Mailbox

200S Boost заключва Overclocking Mailbox — никакви допълнителни manual OC в BIOS или OS след активиране. Или 200S Boost, или Manual OC — не двете едновременно!

1DPC изискване — ти отговаряш технически

200S Boost е проектиран за 1 DIMM per channel с памет до DDR5-8000. Твоите 2×32GB = 1DPC (по 1 DIMM на канал). Технически отговаряш, но официалните тестове са с 2×16GB или 2×24GB. При 2×32GB dual rank може да е по-малко стабилно — тествайте.

Пълна тест последователност

1
POST проверка — без MemOK! beeps. В CPU-Z → Memory tab: проверете честота, timings, voltage съответстват точно.
2
AIDA64 baseline — запишете bandwidth и latency за сравнение.
3
HCI MemTest 200% — ~30–45 мин при 64GB. Fail → не продължавайте, диагностицирайте.
4
TM5 Anta777 Extreme — 1 цикъл — Fail → диагностика. Pass → продължете.
5
y-cruncher 5b — 15 мин — CPU+memory interaction тест.
6
TM5 Anta777 Extreme — 2 цикъла — Gold standard за daily stable.
7
Prime95 Blend (no AVX) — 1 час — Смесено натоварване.
8
Cold boot тест — Изключете PC от контакта за 5 мин, пуснете. Arrow Lake специфика!
9
24h daily use — Gaming, rendering без crash = финална одобрение.

Настройки, които НИКОГА не пипате

  • tRFC под 450 ns при 32GB A-die — data integrity риск
  • 1N Command Rate при 2×32GB DR — практически гарантирана нестабилност
  • Memory Controller Voltage над 1.35V за daily use
  • VCCSA над 1.30V за daily use
  • DRAM VDD над 1.50V за daily use
  • D2D Ratio под Ring Ratio минимум (15×)
  • tRRDL под 8 при Hynix A-die — специфична die слабост
  • Gear 4 при DDR5 под 7600 — наказателна латентност
  • VNNAON над 1.0V за daily use
  • 200S Boost + Manual OC едновременно — несъвместими

DIMM Fit процедура

Кога и как да използвате DIMM Fit

Стартирайте DIMM Fit при стабилна честота преди timing tightening — оптимизира signal integrity за 7–8 часа. Ако планирате 6400 MT/s, първо постигнете стабилност на 6400, после пуснете DIMM Fit, после тегнете timings.

AI Tweaker → DRAM Timing Control → DIMM Fit → [Run] → DIMM Fit Store → [Store0]
